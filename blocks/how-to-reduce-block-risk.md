# Как снизить риск блокировки

## Коротко

Если для вашей связки критична устойчивость, не светите инфраструктуру без необходимости и не смешивайте рабочие домены с лишними рисками.

## Что помогает на практике

1. Не светить origin IP без необходимости.
2. Не держать всё на одной видимой схеме.
3. Не делать лишние резкие перестройки перед запуском.
4. Проверять домен до запуска трафика.

## Что не гарантирует безопасность

Ни один отдельный шаг не даёт абсолютной гарантии. Задача — снизить риск, а не обещать невозможное.
